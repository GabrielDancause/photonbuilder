Image Optimization Guide

A comprehensive guide to optimizing web images for blazing fast performance and improved SEO.

1. Image Formats

Next-Gen

AVIF

The current champion of image compression. Offers 50% better compression than JPEG and 20% better than WebP while maintaining visual fidelity.

Modern Standard

WebP

Universally supported across modern browsers. Typically 25-35% smaller than comparable JPEG or PNG images.

Legacy / Fallback

JPEG

Best used as a fallback for photographs and complex images where older browser support is required.

Legacy / Fallback

PNG

Best used for images requiring transparency or simple graphics with solid colors, though WebP often handles both better.

2. Compression Techniques

Lossy vs. Lossless

Lossy compression reduces file size by permanently removing some data. Tools like MozJPEG, pngquant, or modern WebP/AVIF encoders achieve 70-90% reduction with imperceptible quality loss.

Lossless compression optimizes the file without removing visual data, ideal for technical diagrams or medical imaging, yielding typically 10-20% reduction.

3. Responsive Images

Serve the right sized image based on the user's viewport using the srcset and sizes attributes. This prevents mobile devices from downloading massive desktop-sized images.

<img
  src="image-800w.jpg"
  srcset="image-400w.jpg 400w,
          image-800w.jpg 800w,
          image-1200w.jpg 1200w"
  sizes="(max-width: 600px) 100vw, 50vw"
  alt="Descriptive text">

To serve modern formats with legacy fallbacks, use the <picture> element:

<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Descriptive text">
</picture>

4. Lazy Loading & Decoding

Defer loading off-screen images until the user scrolls near them. Modern browsers support native lazy loading.

<img src="heavy-image.webp" loading="lazy" decoding="async" alt="Background">
  • loading="lazy": Defers the download until the image is near the viewport.
  • decoding="async": Allows the browser to decode the image off the main thread, reducing UI jank.
  • Note: Never lazy-load images above the fold (like hero images), as it will hurt your Largest Contentful Paint (LCP) metric.

5. Content Delivery Networks (CDNs)

Image CDNs (like Cloudinary, Imgix, or Cloudflare Image Resizing) automatically transform, optimize, and cache images at the edge. They detect the user's browser capabilities and serve the most optimal format (e.g., AVIF) automatically.

6. SEO: Alt Text & Sitemaps

Alt Text Best Practices

  • Be descriptive and specific.
  • Keep it concise (under 125 characters).
  • Don't start with "image of" or "picture of".
  • Use keywords naturally, avoid stuffing.
  • Leave empty (alt="") for purely decorative images.

Image Sitemaps

Help Google discover images that might be loaded via JavaScript or CSS by including them in your XML sitemap:

<image:image>
  <image:loc>https://example.com/img.webp</image:loc>
  <image:caption>Image description</image:caption>
</image:image>

Frequently Asked Questions

What is the best image format for websites in 2026?
AVIF and WebP are currently the best formats for websites. AVIF offers the best compression ratios while maintaining high quality, and WebP is universally supported across all modern browsers.
How does image lazy loading affect SEO?
Lazy loading improves SEO by speeding up the initial page load time and improving Core Web Vitals (specifically Largest Contentful Paint). Search engine bots can still discover lazy-loaded images.
Should I still use JPEG and PNG images?
You should generally serve WebP or AVIF to modern browsers, but keep JPEG and PNG as fallbacks for older browsers using the <picture> element.
What is lossy vs lossless compression?
Lossy compression removes some data from the image to reduce file size significantly, often without a noticeable drop in visual quality. Lossless compression reduces file size without losing any data, but the file size reduction is much smaller.
How do I write good alt text for SEO?
Good alt text should be descriptive, concise, and contextually relevant. Avoid keyword stuffing, and focus on describing the image for someone who cannot see it.
