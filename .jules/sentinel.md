## 2025-03-16 - Prevent XSS in Template Literals within Astro Pages
**Vulnerability:** Several client-side scripts inside Astro pages directly inject unsanitized variables (like user data, error messages, or JSON fields) into the DOM via \`.innerHTML\` using template literals (e.g., \`tr.innerHTML = \\`<td>${c.company}</td>\\`\`).
**Learning:** Even though Astro strictly sanitizes server-side renders and bindings (e.g. \`{variable}\`), client-side scripts injected directly into \`<script>\` blocks are executed in the browser. When generating interactive tables/components using Vanilla JS on the frontend, using \`.innerHTML\` without a sanitizer exposes the page to Cross-Site Scripting (XSS).
**Prevention:** Avoid \`.innerHTML\` where possible (prefer \`document.createElement\` and \`.textContent\`). If \`.innerHTML\` must be used for layout performance or string templating, always define an \`escapeHTML()\` utility function and securely wrap all interpolated variables inside the literal (e.g., \`tr.innerHTML = \\`<td>${escapeHTML(c.company)}</td>\\`\`).
