When to use chmod 755

Use 755 for: directories (so users can list and enter them), shell scripts and executables, web server directories (like /var/www), and any file or folder that needs to be publicly accessible but only editable by the owner.

Security note: 755 is generally safe. The only consideration is that everyone can read the file contents and execute it. For sensitive scripts containing credentials, use 700 instead.

Consider instead: If group members should also edit: use 775. If nobody else should access it: use 700. For non-executable files like HTML or images: use 644.

Common Mistake

Setting regular files (like .html, .css, .jpg) to 755. These files don't need execute permission — use 644 instead. Execute permission on non-scripts is unnecessary and slightly less secure.

chmod 755 vs 750

755 vs 750
755 vs 750: The difference is that 750 blocks 'others' (users not in the file's group) from any access. Use 750 when you want the owner's group to access the file but not random other users on the system.

Frequently Asked Questions

When should I use chmod 755?
Use 755 for directories, executable scripts, and program files. It's the default for most system directories and the correct permission for web server document roots.
What's the difference between 755 and 644?
755 includes execute permission, 644 doesn't. Use 755 for directories and scripts, 644 for regular files like documents, images, and config files.
Should web files be 755 or 644?
Directories should be 755 (so the web server can enter them). Regular files (.html, .css, .js, images) should be 644. Only CGI scripts or server-side executables need 755.
