Frequently Asked Questions

What is the average web page size in 2026?

Based on our measurement of 100 top websites, the average web page size is 2.2 MB (2,217 KB), with a median of 1.5 MB. This includes the HTML document, all linked CSS stylesheets, JavaScript files, and discoverable images. The range spans from under 1 KB (redirect pages) to over 13 MB (YouTube).

How much JavaScript does the average website load?

JavaScript is typically the largest component of modern web pages. In our study, YouTube loads 9.7 MB of JavaScript alone, Discord loads 7.1 MB, and Spotify loads 6.6 MB. For sites where we could measure sub-resources, JavaScript accounts for 60–80% of the total page weight.

Which websites are the heaviest?

YouTube (13.2 MB), HBO Max (10.5 MB), and Discord (10.1 MB) are the heaviest sites in our study. These are JavaScript-heavy web applications that load significant client-side code for their interactive features.

Which industry has the heaviest websites?

Gaming and entertainment sites tend to be the heaviest due to rich media and interactive features. Media/news sites are also above average due to ad tech, tracking scripts, and high-resolution images. Finance sites vary widely — some are lightweight marketing pages while others load heavy banking applications.

How is page weight measured?

Page weight is the total bytes downloaded to fully render a page. We measured it by fetching each site's HTML, parsing it for linked resources (CSS, JS, images via <link>, <script>, and <img> tags), then making HTTP HEAD requests to each resource URL to get its Content-Length header. The sum of all resource sizes equals the page weight. This method may undercount pages that load resources dynamically via JavaScript.

Methodology

Sample: 100 of the most visited websites globally across 7 industries: tech, media, e-commerce, finance, social media, travel, and gaming/entertainment. Sites were selected based on traffic rankings and industry representation.

Method: For each site, a Node.js script made an HTTPS GET request to the root URL, then parsed the HTML response to extract all linked resource URLs (CSS via <link rel="stylesheet">, JavaScript via <script src>, images via <img src>, and preloaded resources via <link rel="preload">). For each discovered resource, a HEAD request was made to obtain the Content-Length header. Total page weight = HTML body size + sum of all resource Content-Length values.

Limitations: This method measures server-reported sizes, not actual transfer sizes (which may differ with compression). Resources loaded dynamically via JavaScript after initial page load are not counted. Some CDNs block HEAD requests, resulting in 0-byte measurements for CSS/JS on approximately 14% of sites. Image discovery is limited to <img> tags in the HTML — CSS background images and lazy-loaded images are not counted.

Reproducibility: The crawl script and raw JSON data are available in our GitHub repository. Data collected March 8, 2026.
